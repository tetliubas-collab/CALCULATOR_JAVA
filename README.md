# ☕ Java Calculator — Fullstack (Vue 3 + Spring Boot)

Повноцінний fullstack калькулятор:
- **Бекенд:** Java 17 + Spring Boot (порт 3000) — REST API
- **Фронтенд:** Vue 3 + Vite (порт 5173) — SPA інтерфейс

---

## ⚙️ Вимоги

| Інструмент | Версія |
|---|---|
| Java | 17+ |
| Maven | 3.8+ (або використовуй IDE) |
| Node.js | 18+ |
| npm | 9+ |

Перевірити: `java -version` / `node -v` / `npm -v`

---

## 🚀 Запуск проекту

### 1. Запустити Java бекенд

```bash
cd backend
mvn spring-boot:run
```
Бекенд стартує на: `http://localhost:3000`

### 2. Запустити Vue фронтенд (окреме вікно терміналу)

```bash
cd frontend
npm install
npm run dev
```
Фронтенд відкривається на: `http://localhost:5173`

## 📋 Лабораторна №3 — Виконані кроки

### Крок 1: Скрипти автоматизації (package.json)

```bash
npm run dev        # запуск сервера розробки (Vite)
npm run build      # продуктова збірка → папка dist/
npm run preview    # локальний запуск збірки dist/
npm run lint       # ESLint — автовиправлення помилок
npm run lint:check # ESLint — тільки перевірка (без виправлень)
npm run format     # Prettier — форматування коду
```

### Крок 2: Змінні оточення

Файл `.env` (Development):
```
VITE_APP_STATUS=Development
VITE_API_URL=http://localhost:3000
VITE_APP_TITLE=Java Calculator
```

Файл `.env.production` (Production):
```
VITE_APP_STATUS=Production Mode
VITE_API_URL=http://localhost:3000
VITE_APP_TITLE=Java Calculator
```

**Де відображається:** у хедері (`env-badge`) та футері застосунку.
- `npm run dev` → показує синій бейдж **"Development"**
- `npm run build && npm run preview` → показує зелений бейдж **"Production Mode"**

### Крок 3: ESLint

```bash
cd frontend

# Перевірити наявність помилок
npm run lint:check

# Автовиправлення
npm run lint
```

Конфігурація: `eslint.config.js`
- Заборона невикористаних змінних (`no-unused-vars`)
- Попередження про `console.log` у коді
- Одинарні лапки, без крапки з комою

### Крок 4: Генерація та аналіз артефакту

```bash
cd frontend
npm run build
```

Результат у папці `dist/`:
```
dist/
├── assets/
│   ├── index.[HASH].css    ← мініфікований CSS
│   └── index.[HASH].js     ← мініфікований JS бандл
└── index.html
```

**Хеші у назвах файлів** (наприклад `index.R4VXs7b2.js`):
- Дозволяють браузеру кешувати файли нескінченно
- При зміні коду хеш змінюється → браузер завантажує нову версію
- Без хешів браузер міг би показувати застарілу кешовану версію

**Порівняння розмірів (орієнтовно):**

| Папка | Розмір | Опис |
|-------|--------|------|
| `src/` | ~25 KB | Сирцевий код (читабельний, з коментарями) |
| `dist/` | ~8 KB | Мініфікований бандл (gzip: ~3 KB) |

### Крок 5: Перевірка збірки

```bash
npm run build
npm run preview
# Відкрити http://localhost:4173
```

Перевірити що в Production Mode:
- Бейдж змінився на зелений "Production Mode"
- Калькулятор працює коректно
- Усі операції (+, −, ×, ÷, %, ^, √) функціонують

### Крок 6: Гігієна репозиторію

`.gitignore` містить:
```
node_modules/   ← не потрібно у репо (занадто великий, ~200 МБ)
dist/           ← генерується командою build
.env            ← локальні налаштування (можуть містити секрети)
```

Git відстежує:
```
eslint.config.js     ← критично для автоматизації
.prettierrc          ← стиль коду команди
.prettierignore      ← що не форматувати
.env.production      ← безпечні налаштування продакшену
```

## ❓ Відповіді на контрольні запитання

**1. Чому dist/ у .gitignore?**
Папка `dist/` — це результат збірки, який генерується автоматично командою `npm run build`. Зберігати її у Git немає сенсу, оскільки вона великого розміру та завжди може бути відтворена з сирцевого коду. Крім того, бінарні мініфіковані файли погано відображаються у `git diff`.

**2. Навіщо хеш у назвах файлів?**
Хеш (наприклад `index.R4VXs7b2.js`) — це відбиток вмісту файлу. Браузер кешує файл за його назвою. Якщо назва не змінилася — браузер використовує кеш. При зміні коду хеш змінюється → нова назва → браузер завантажує свіжу версію. Це дозволяє одночасно мати "нескінченний" кеш та завжди отримувати актуальний код.

**3. dependencies vs devDependencies?**
`dependencies` (vue) — потрібні для роботи застосунку у браузері, входять у фінальний бандл.
`devDependencies` (vite, eslint, prettier) — тільки для розробки та збірки, у продакшн бандл НЕ входять. Це зменшує розмір фінального артефакту.

**4. Що станеться без збірки?**
Браузер не вміє працювати з `.vue` файлами напряму. Також не буде мініфікації (великий розмір), транспіляції (проблеми у старих браузерах), tree shaking (зайвий код). Застосунок просто не запуститься.
